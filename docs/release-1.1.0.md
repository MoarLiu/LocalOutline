# Local Outline 1.1.0 更新说明

发布日期：2026-05-22

## 重点更新

- 新增 Markdown 文档模式，和“大纲 / 脑图 / 演示”同级，可在源码、预览、分栏三种布局间切换。
- Markdown 模式优先保留原始 Markdown 文本；切换到大纲、脑图或演示时会同步解析成现有节点树。
- 新增 Markdown 常用格式工具栏，支持标题、加粗、斜体、删除线、行内代码、引用、列表、任务、链接、图片、代码块和表格。
- 全局拦截 `Ctrl+S` / `Command+S`，不再触发浏览器“保存网页”，而是先刷新当前草稿并保存到本地工作区。
- “最近编辑”从占位入口升级为可用文档列表，并显示最近编辑时间。

## 体验优化

- Markdown 模式隐藏右侧节点详情面板，让源码编辑区和预览区使用完整工作区宽度。
- 大纲编辑区最大宽度提升到约 1040px，长主题输入时文本列空间更充足。
- 脑图节点支持更长单行文本，并调整横向间距，降低长标题和连线挤压。
- 顶部“大纲 / 脑图 / 演示 / Markdown”切换按钮保持单行显示。
- Markdown 预览支持多层列表、任务列表、代码块、表格、图片和基础行内格式。
- 工具条中的 Markdown 导出入口改名为“导出 MD”，避免和 Markdown 编辑模式混淆。

## 数据与兼容性

- 文档数据增加 `markdownSource` 和 `markdownUpdatedAt` 可选字段，工作区版本号保持兼容。
- 从 Markdown 解析节点时会尽量复用已有节点 ID 和折叠、颜色、样式、图标等状态，减少视图切换抖动。
- 当用户在结构化视图继续编辑节点、标题、备注或样式时，会以节点树为准重新生成 Markdown，避免原文和结构内容长期不一致。
- Markdown 图片会保留 alt 和 URL；表格单元格中的 `|` 会进行转义处理。
- OPML 导出增加 Local Outline 扩展属性，用于保留更多节点状态。

## 修复

- 修复 Markdown 中第二个及后续 `# H1` 被解析时丢失的问题。
- 修复空 Markdown 原文导出时被错误重生成的问题。
- 修复 Markdown 模式顶部标题编辑可能破坏原文同步的问题。
- 修复导入和载入备份前未刷新 Markdown 草稿，导致最后输入丢失的风险。
- 修复 Markdown 纯预览模式下点击编辑工具会误写到文档末尾的问题。
- 修复移动端 Markdown 编辑区仍扣除侧栏宽度导致过窄的问题。
- 修复最近编辑相对时间停留过久不刷新的问题。
